# Upseller 后台操作与面单排序技能

## 技能目标
帮助用户系统化操作 Upseller 后台，解决订单处理、面单打印、商品分拣、批量发货、标签排序、爆款打包机配合等问题。

核心目标：
让订单和面单能尽量按“单品单样 / SKU集中 / 同款连续”的方式排序，方便仓库快速配货、贴单、打包，降低错发和漏发。

---

## 安全边界

### 不允许
- 不保存 Upseller 账号密码
- 不把账号密码写入代码
- 不自动登录资金、付款、退款等高风险页面
- 不自动批量执行不可逆操作
- 不自动删除订单、取消订单、退款

### 允许
- 用户自己在紫鸟浏览器或本机浏览器登录 Upseller
- 系统根据截图指导用户点哪里
- 用户导出订单 Excel/CSV 后，系统分析排序
- 系统生成推荐排序规则
- 系统生成可导入/可打印的订单排序表
- 如果未来做浏览器自动化，只能在用户已登录环境下执行低风险动作，并在打印/发货前二次确认

## 业务问题定义

用户当前痛点：
每次打印面单时，Upseller 排序无法做到单品单样集中打印，导致同一个 SKU 的面单分散，无法配合爆款打包机高效率处理包裹。

理想结果：
1. 同一个 SKU 的订单连续排列
2. 单品单件订单优先集中打印
3. 多品多件订单单独分区处理
4. 爆款 SKU 可单独批量打印
5. 面单顺序和拣货顺序一致
6. 打包人员按一摞面单即可连续处理同款商品

---

## 标准订单分类

### A类：单品单件订单
一个订单只有一个 SKU，数量为1。

优先级最高，最适合爆款打包机连续处理。

### B类：单品多件订单
一个订单只有一个 SKU，但数量大于1。

可放在同SKU后面处理，但需要标记数量。

### C类：多品订单
一个订单包含多个 SKU。

不建议混在爆款批量打包流程里，应单独打印或单独拣货。

### D类：异常订单
地址异常、缺货、待确认、备注异常、买家留言、物流异常。

必须单独处理，不参与批量打印。

---

## 推荐排序逻辑

优先排序字段：
1. 订单类型：A单品单件 → B单品多件 → C多品 → D异常
2. SKU编码
3. 商品名称
4. 颜色/规格
5. 数量
6. 订单时间
7. 物流渠道

最终目标：
相同 SKU 的面单连续打印。

---

## Upseller 后台尝试路径

需要依次检查后台是否支持以下功能：

1. 订单列表是否能按 SKU 排序
2. 订单列表是否能按商品名称排序
3. 订单列表是否能筛选单品订单
4. 是否能导出待发货订单
5. 是否能批量导入排序后的订单号
6. 是否能按勾选顺序打印面单
7. 是否能按订单号顺序打印面单
8. 是否能创建拣货单/配货单
9. 是否能生成商品汇总拣货单
10. 是否能按商品分组打印

如果后台本身不支持单品单样排序，则采用导出订单 → 本地排序 → 按排序结果分批勾选打印的方案。

---

## 解决方案路径

### 方案1：后台内置排序
如果 Upseller 支持按 SKU/商品名排序：
1. 进入待发货订单
2. 选择需要打印的订单
3. 按 SKU 或商品名排序
4. 先筛选单品单件订单
5. 批量打印面单
6. 再处理多品订单

### 方案2：导出订单表排序
如果后台不能直接按 SKU 打印：
1. 从 Upseller 导出待发货订单 Excel/CSV
2. 系统读取订单表
3. 自动识别：单品单件、单品多件、多品订单、异常订单
4. 按 SKU 排序生成打印顺序
5. 输出订单号排序清单
6. 用户按排序清单在后台分批勾选打印

### 方案3：爆款 SKU 单独打印
适合爆款商品：
1. 找出当天订单量最高 SKU
2. 筛选该 SKU 的单品单件订单
3. 单独打印该 SKU 面单
4. 配合爆款打包机连续打包
5. 多品订单最后人工处理

### 方案4：开发辅助工具
让 Codex 开发一个本地网页工具：
- 上传 Upseller 导出的订单表
- 自动识别 SKU
- 自动分类订单
- 自动生成打印批次
- 自动生成拣货单
- 自动生成面单打印顺序
- 标记多品订单和异常订单

---

## 本地工具功能要求

Codex 应开发：

### 1. 订单导入
支持 Excel/CSV 上传。

### 2. 字段识别
自动识别或手动映射：
- 订单号
- SKU
- 商品名称
- 规格
- 数量
- 买家备注
- 物流渠道
- 订单状态

### 3. 订单分类
自动分成：
- 单品单件
- 单品多件
- 多品订单
- 异常订单

### 4. SKU排序
按 SKU、商品名、规格、数量排序。

### 5. 打印批次生成
输出：
- 第1批：爆款SKU A
- 第2批：爆款SKU B
- 第3批：其他单品单件
- 第4批：单品多件
- 第5批：多品订单
- 第6批：异常订单

### 6. 拣货单
生成商品汇总：
- SKU
- 商品名称
- 数量
- 对应订单数

### 7. 面单顺序表
生成：
- 打印顺序
- 订单号
- SKU
- 商品
- 数量
- 备注
