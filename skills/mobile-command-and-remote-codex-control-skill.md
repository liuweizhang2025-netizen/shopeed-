# 手机端指令与远程 Codex 控制技能

## 技能目标
让用户可以通过手机端 ChatGPT 或 Codex 远程下达电商运营指令，由电脑端、云端或仓库中的系统继续执行任务，实现移动端随时指挥、查看、确认和调整。

核心目标：
用户不一定坐在电脑前，也能通过手机完成：
- 发出任务
- 查看进度
- 批准方案
- 暂停任务
- 修改方向
- 让 Codex 继续开发
- 让系统继续分析 Shopee/Upseller/1688 数据

---

## 使用场景
当用户说：
- 手机端操作
- 手机指令控制
- 用手机让 Codex 跑
- 我在外面也要指挥系统
- 远程操作
- 移动端下命令
- 手机端 ChatGPT 控制电脑任务

即调用本技能。

---

## 核心原则

1. 手机端负责发指令和确认。
2. Codex/电脑端/网页工具负责执行开发或分析。
3. 高风险后台操作必须二次确认。
4. 不在手机端直接暴露或保存账号密码。
5. 遇到验证码、登录过期、付款、退款、批量改价等动作，必须暂停并请求用户确认。
6. 每个任务必须有状态：待执行、执行中、等待确认、已完成、失败、已暂停。

---

## 手机端可以做什么

### 允许
- 创建任务
- 修改任务方向
- 查看 Codex 开发结果
- 审核 PR 或代码改动
- 让系统分析截图/表格/PDF
- 发 Shopee 广告截图让系统判断
- 发买家消息让系统生成客服回复
- 发产品图让系统生成主图方案
- 发 Upseller 页面截图让系统指导操作
- 远程批准低风险代码提交

### 谨慎
- 批量改标题
- 批量改价格
- 批量设置折扣
- 批量打印面单
- 批量回复客户
- 调整广告预算

### 禁止默认自动做
- 自动付款
- 自动退款
- 自动提现
- 自动保存账号密码
- 自动绕过验证码
- 自动大批量删除商品
- 自动无确认发货
- 自动无确认修改广告预算

## 手机端指令格式

建议用户每次用手机发指令时，按以下格式：

任务：xxx
目标：xxx
资料：截图/链接/表格/PDF
限制：不要自动执行高风险操作
输出：我需要xxx
确认：执行前先问我

---

## 示例指令

### 1. 远程让 Codex 开发
任务：让 Codex 开发 Upseller 订单排序工具
目标：上传订单Excel后自动按SKU分组，生成打印批次
限制：不要连接真实账号，不保存密码
输出：给我网页工具和README
确认：提交代码前先让我看

### 2. 手机发广告截图
任务：分析今天3店Shopee广告
目标：判断哪些SKU亏损、哪些继续跑
资料：广告截图
输出：预算、ROAS、暂停建议
确认：不要自动改后台，只给建议

### 3. 手机发客服消息
任务：回复Shopee买家
目标：生成巴葡客服回复
资料：买家消息截图
输出：巴葡回复+中文意思+风险判断
确认：不要自动发送

### 4. 手机发主图需求
任务：生成Shopee主图方案
目标：手机端高点击高转化
资料：产品图+竞品图
限制：不能改变产品结构
输出：主图Prompt和自检

## 远程任务状态机制

每个任务必须有状态：

1. 待执行
2. 执行中
3. 等待用户确认
4. 等待验证码/登录处理
5. 已完成
6. 失败
7. 已暂停

---

## 遇到验证码/风控时

如果手机端收到提示：
- Shopee验证码
- Upseller验证码
- 1688验证码
- 登录过期

系统必须：
1. 暂停任务
2. 发截图或说明给用户
3. 告诉用户需要人工验证
4. 用户处理完成后说“好了”
5. 系统继续原任务

不得绕过验证码。

---

## Codex 手机端协作方向

Codex 可以通过 ChatGPT 移动端远程查看任务、发起任务、审核输出、批准修改和管理代码任务。

Codex 的重点不是直接操作 Shopee 后台，而是：
- 开发本地工具
- 修改代码
- 修Bug
- 生成网页系统
- 写自动化脚本
- 生成数据分析工具
- 做PR和版本管理

后台业务操作必须由用户确认或半自动执行。

## 手机端优先开发的功能

### 1. 任务看板
手机上能看到：
- 今天广告待分析
- SKU待巡检
- Upseller订单待排序
- 客服待回复
- 主图待优化
- 选品待验证

### 2. 一句话指令入口
例如：
- 分析今天广告
- 生成客服回复
- 检查这个SKU
- 做主图
- 排序今天订单

### 3. 确认按钮机制
高风险任务必须显示：
- 建议动作
- 风险
- 是否执行

用户确认后才继续。

### 4. 日报推送
每天输出：
- 亏损SKU
- 赚钱SKU
- 需要改图SKU
- 需要暂停广告SKU
- 需要处理差评SKU
- 待打印订单批次

## 自检规则

每次执行前检查：
1. 是否适合手机端指令
2. 是否需要电脑端执行
3. 是否涉及高风险后台操作
4. 是否需要用户二次确认
5. 是否保护账号密码
6. 是否能中断后恢复
7. 是否有明确任务状态
